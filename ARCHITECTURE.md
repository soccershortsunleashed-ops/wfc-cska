# Архитектура wfc-cska

## Обзор

Проект — сайт ЖФК ЦСКА на Next.js (App Router) с API-слоем для динамических данных, Prisma для доступа к БД и отдельным набором утилит/скриптов для загрузки и синхронизации контента.

Ключевая особенность: **статистика матчей и игроков синхронизируется с сервисом Rustat** и сохраняется в БД для дальнейшего отображения на сайте.

## Слои

1. **UI и страницы**
   - `app/` — маршруты страниц и серверные компоненты.
   - `components/` — UI-компоненты, секции, блоки страниц.

2. **API и сервисы**
   - `app/api/` — API-эндпойнты для новостей, матчей, игроков, таблиц.
   - `lib/services/` — бизнес-логика и доступ к данным.
   - `lib/services/rustat-*.ts` — интеграция и запросы к Rustat.

3. **Данные и БД**
   - `prisma/` — схема БД, миграции, сиды.
   - `seed/` — локальные наборы данных (новости, составы, логотипы и т.п.).

4. **Инструменты и синхронизация**
   - `scripts/` — утилиты импорта, проверки и синхронизации (включая Rustat).

## Поток данных (кратко)

1. Скрипты в `scripts/` загружают/обновляют данные и синхронизируют статистику с Rustat.
2. Данные сохраняются в БД через Prisma.
3. API-эндпойнты в `app/api/` используют сервисы из `lib/services/`.
4. Страницы в `app/` запрашивают данные через сервисы или API и рендерят UI.

## Источник статистики

- **Rustat** — основной источник матчевой и персональной статистики игроков.
- Скрипты синхронизации и проверки находятся в `scripts/` (например, `rustat-*.mjs`, `sync-*.mjs`).
- Интеграция реализована в `lib/services/rustat-*.ts`.

## Развертывание

- Рекомендуемый провайдер: Vercel.
- Альтернативы: Railway, Render, DigitalOcean App Platform.
- Детали см. в `DEPLOYMENT.md`.

## Как развивать

- Расширять схемы Prisma и миграции под новые данные.
- Добавлять/обновлять интеграции в `lib/services/`.
- Поддерживать скрипты синхронизации и проверки данных.
